# Scope Tiers Specification — The Product Systems Engine

This document establishes the official functional hierarchy for the portfolio project, ensuring a strict guardrail against scope drift and timeline dilation.

## 1. Tier 1: The Must-Have (MVP Baseline)
The MVP tier represents the core conversion engine. If WebGL loading fails, or if a user visits via a low-tier mobile client, this tier guarantees absolute brand positioning and communication of technical authority.

* **Core Deliverables:**
  * Fully responsive Next.js App Router structural layout with a graphite aesthetic.
  * All 10 story sections mapped natively as clean, semantic HTML sections.
  * Core identity blocks, project detail layouts, and structural case-study templates.
  * Accessible contact CTA modules located in the hero and footer sections.
* **Performance Budget Baseline:**
  * Lighthouse Performance Score: 95+ (Canvas layer omitted).
  * Initial Page Content Visibility: Under 1.0s over standard connections.
* **Accessibility Control:**
  * 100% semantic structure with explicit keyboard focus paths.

## 2. Tier 2: The Should-Have (Enhanced 3D Experience)
The Enhanced tier activates the immersive 3D layer. It implements the scroll-bound storytelling arc using highly optimized assets.

* **Core Deliverables:**
  * Persistent React Three Fiber (R3F) canvas managing global state transitions across scroll coordinates.
  * Smooth, reversible camera paths linking the 10 storytelling vectors seamlessly.
  * Low-poly system geometries (assemblies, mesh networks, structural architecture plates).
  * Strict asset payload caps: Max 3MB total compressed GLB payload.
* **Performance Budget Baseline:**
  * Target Frame Rate: Stable 55-60 FPS on mid-tier desktop setups.
  * Max Triangles: Under 150k visible per scene.

## 3. Tier 3: The Could-Have (Premium Polish)
The Premium tier encapsulates advanced visual adjustments that elevate the site's cinematic feel without affecting its informational core.

* **Core Deliverables:**
  * Custom GLSL fragment shaders for dynamic mesh interactions (e.g., precise network data transmission animations).
  * Advanced real-time contact layout illumination pulses linked directly to mouse hover vectors.
  * Depth-of-field post-processing adjustments (only if desktop frame rates remain within budget parameters).
* **De-risking Rule:**
  * Automatically stripped if client initialization detects device hardware constraints or reduced-motion preferences.
