# Geometry Vocabulary Specification — The Product Systems Engine

This specification codifies the allowed geometric laws, low-poly structural mesh boundaries, and forbidden visual clichés for the 3D canvas layer.

## 1. Approved Geometric Vocabulary (The Allowed Shapes)
Every visible 3D object must map directly to software engineering systems. All meshes utilize clean low-polygon distributions with crisp `#1a1a1a` wireframe contours.

* **Concentric Structural Rings (The Runtime Engine Loops):** Thin circular shapes representing core processing loops, runtime threads, and initialization sequences.
* **Stratified Layer Plates (The Stacking Application Layers):** Flat, horizontal bounding planes representing independent systemic blocks (such as separating UI views from database connections).
* **Interconnected Vector Nodes (The Component Dependency Network):** Grid lines intersecting at sharp vertices, representing modular component connection webs, API endpoints, and conditional statement trees.
* **Monolithic Capsule Blocks (The Product System Manifests):** Uniform solid structural boxes representing compiled production-ready applications housing protected internal mechanics.

## 3. Visual Non-Goals (The Forbidden Clichés)
To protect the premium brand alignment, sub-agents are strictly prohibited from implementing the following decorative elements:

* **No Hacker Cyberpunk Noise:** Absolutely no falling digital code rain, scrolling terminal overlays, neon glowing circuit tracks, or digital skulls.
* **No Abstract Gaming Objects:** No floating plasma vectors, organic fluid meshes, gaming avatars, or detached floating terrain maps.
* **No Over-Texturing:** Avoid multi-layered high-resolution image texturing. All shapes rely on solid graphite color variables, minimal roughness tokens, and precise wireframe outlines to guarantee rapid payload delivery.
